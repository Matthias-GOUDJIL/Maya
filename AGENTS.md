# AGENTS.md - AI Agent Guide for Maya MMO

This file provides essential information for AI coding assistants working on the Maya MMO project.

## Project Overview

Maya is a fantasy sandbox MMO built with:
- **Client**: Godot 4.6+ (GDScript)
- **Server**: Node.js + TypeScript
- **Shared**: Protocol definitions and types
- **Database**: PostgreSQL (persistent) + Redis (real-time)


## Coding Conventions

### GDScript (Client)
- Use `snake_case` for variables and functions
- Use `PascalCase` for class names and node names
- Always use `extends` to specify parent class
- Use typed GDScript when possible (`var x: int = 0`)
- Signal connections use `signal_name.connect(_on_signal_name)`

### TypeScript (Server)
- Use `camelCase` for variables and functions
- Use `PascalCase` for classes, interfaces, and types
- Use strict typing - avoid `any`
- Use ECS (Entity Component System) architecture

### Scene Files (.tscn)
- **NEVER** use `.new()` in scene files
- Use `[sub_resource]` blocks for inline resources
- Always provide 4 arguments to `Color()` (RGBA)
- Use `ExtResource()` for external resources, `SubResource()` for internal
- `load_steps` must match the actual number of external + sub resources

### Shader Files (.gdshader)
- Godot 4 CanvasItem/Sky/Particles shader format
- Use `hint_color` for color uniforms

## Common Tasks

### Adding a New Scene
1. Create `.tscn` file in appropriate `scenes/` subdirectory
2. Create corresponding `.gd` script in `scripts/`
3. Never use `.new()` in `.tscn` files
4. Use proper `load_steps` count

### Adding a New Server System
1. Create `.ts` file in `server/src/systems/`
2. Implement system logic following ECS pattern
3. Register system in main game loop

### Modifying Protocol
1. Update `shared/protocol/Protocol.ts`
2. Update corresponding server system
3. Update client script if needed

## File Formats Reference

### Valid .tscn Resource Creation
```
[sub_resource type="PlaneMesh" id="PlaneMesh_ground"]
size = Vector2(100, 100)

[node name="Ground" type="MeshInstance3D"]
mesh = SubResource("PlaneMesh_ground")
```

### Invalid (will cause parse errors)
```
mesh = PlaneMesh.new()  # DON'T DO THIS
```

## Git Workflow
- `.godot/` folder is ignored (Godot cache)
- `.gitignore` is configured for Godot + Node.js
- Always run lint/typecheck before committing
- For Godot: Use built-in script editor warnings
- For TypeScript: `cd server && npm run lint && npm run typecheck`

## Important Notes

1. **No `.new()` in .tscn files** - This is the most common error
2. **Color() needs 4 args** - RGBA format required in scene files
3. **`preload()` not valid in .tscn** - Use `ExtResource` instead
4. **`.uid` files** are auto-generated by Godot 4 - commit them
5. **`.import` files** are auto-generated - commit them
6. **Server `dist/`** folder is committed (compiled JS for deployment)

## Troubleshooting

### Godot Scene Parse Errors
- Check for `.new()` calls in .tscn files
- Verify `Color()` has 4 arguments
- Check `load_steps` matches resource count
- Ensure no `preload()` in scene files

### Server TypeScript Errors
- Run `npm run build` to check compilation
- Check `tsconfig.json` for proper configuration
